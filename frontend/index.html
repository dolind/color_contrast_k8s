Move the slider to simulate concurrent users. Higher load → higher CPU → Kubernetes scales pods.

- Inflight: Number of requests currently being processed by the backend. Real-world analogy: users actively waiting for a response (concurrent active users).
- Completed: Requests that finished successfully during the last interval (~1s). Real-world analogy: users who received a completed response.
- Errors: Requests that failed (HTTP error or network error). Real-world analogy: users who experienced a failure instead of a valid response.
- Pods: Number of backend instances currently running (what the HPA scales).
